## 1. 项目概述

### 1.1 背景

ComicManager v0.2.0 是一款 Python 桌面应用程序，用于将包含漫画图像的 CBZ（Comic Book ZIP）和 ZIP 文件合并为单个 CBZ 文件。当前实现使用 Tkinter 作为图形界面，在视觉吸引力、响应速度和可维护性方面存在局限。

ComicManager Neo 是一次完全重写，采用现代 Web 技术构建前端，同时保留并扩展所有现有核心功能。

### 1.2 目标

- 使用现代、响应式的基于 Web 的 UI（Vue 3 + TailwindCSS + DaisyUI）替换 Tkinter GUI，运行于 PyWebView 内
- 替换仅支持命令行的后端，采用基于 FastAPI 的本地 HTTP 服务器
- 保留 v0.2.0 的所有现有功能
- 通过更好的视觉设计和交互提升用户体验
- 建立清晰、可维护的架构以支持未来功能扩展

### 1.3 范围

**范围内（v1.0）：**

- 将所有 v0.2.0 功能完整迁移至新技术栈
- CBZ 和 ZIP 文件合并
- 带格式过滤的 ZIP 图像提取
- 基于章节的智能重命名
- 拖放式文件管理
- 带实时更新的进度跟踪
- 设置与配置管理（YAML 格式）
- 通过 PyInstaller 打包为独立可执行文件

**范围外（v1.0）：**

- 在线/云存储集成
- 移动端支持
- 插件系统
- 图像格式转换
- ComicInfo.xml 可视化编辑器
- 批量目录处理

## 4. API 设计

### 4.1 端点

```
基础地址：http://localhost:{动态端口}

# 文件队列管理
POST   /api/files              添加文件到合并队列（多部分表单）
GET    /api/files              获取所有队列文件及其元数据
DELETE /api/files              按索引移除文件（JSON 体：{indices: [0,1,2]}）
PUT    /api/files/reorder      重新排序文件（JSON 体：{from: 0, to: 3}）
POST   /api/files/clear        清空队列中的所有文件
POST   /api/files/validate     验证所有队列文件

# 合并操作
POST   /api/merge              启动合并操作
GET    /api/merge/{任务ID}/progress  SSE 合并进度流

# 浏览（操作系统级文件对话框）
POST   /api/browse/files       打开操作系统文件选择器对话框
POST   /api/browse/directory   打开操作系统目录选择器对话框

# 设置
GET    /api/settings           获取当前设置
PUT    /api/settings           更新设置（JSON 体）

# 格式
GET    /api/formats            获取支持的图像格式
```

### 4.2 数据模型

```python
# 队列文件表示
class QueuedFile(BaseModel):
    path: str              # 绝对文件路径
    name: str              # 文件名
    type: Literal["CBZ", "ZIP"]
    size: int              # 文件大小（字节）
    page_count: int        # 图像数量
    valid: bool            # 验证状态
    error: str | None      # 验证错误（如有）

# 合并请求
class MergeRequest(BaseModel):
    output_filename: str
    output_dir: str
    preserve_metadata: bool = True
    zip_formats: list[str] = ["jpg"]

# 合并进度（SSE 事件）
class MergeProgressEvent(BaseModel):
    task_id: str
    stage: Literal["validating", "extracting", "merging", "writing", "done", "error"]
    current_file: str | None = None
    current_index: int = 0
    total_files: int = 0
    current_page: int = 0
    total_pages: int = 0
    message: str

# 合并结果
class MergeResult(BaseModel):
    success: bool
    output_path: str | None = None
    total_pages: int = 0
    merged_files: list[MergedFileInfo] = []
    errors: list[str] = []

# 应用设置
class AppSettings(BaseModel):
    last_output_dir: str
    last_input_dir: str = ""
    zip_image_formats: list[str] = ["jpg"]
    theme: str = "light"
    preserve_metadata: bool = True
    auto_increment: bool = True
```

## 8. 非功能需求

### 8.1 性能

- 文件验证应在每个文件 2 秒内完成（典型 CBZ/ZIP 不超过 500MB）
- 合并期间 UI 必须保持响应（后台任务 + SSE）
- 启动时间不应超过 3 秒

### 8.2 兼容性

- Windows 10/11（主要目标）
- Python 3.10.8+
- v1.0 不支持 macOS/Linux（PyWebView 行为差异显著）

### 8.3 安全性

- 服务器仅绑定到 127.0.0.1（仅本地主机）
- 所有文件输入的路径遍历验证
- 无需凭据存储（无用户账户）
- 无外部服务网络访问

### 8.4 可靠性

- 临时文件必须在合并完成或失败时清理
- 优雅处理损坏的 ZIP/CBZ 文件
- 无数据丢失：绝不修改原始文件

## 10. 开放问题

1. **PyWebView 拖放**：PyWebView 的文件拖放支持因平台而异。需要验证 HTML5 拖放在 Windows 上配合 PyWebView 6.x 是否正常工作。备用方案：仅使用文件选择器按钮。

2. **SSE 与 WebSocket**：SSE 更简单，适合进度更新。WebSocket 增加复杂性。决定：从 SSE 开始，仅当后续需要双向通信时才切换到 WebSocket。

3. **PyInstaller 单文件与单目录**：单文件更便于分发但启动较慢。决定：通过规范文件提供两种选项。

4. **打包的前端资源**：对于 PyInstaller 构建，构建的 Vue 前端（dist/）应由 FastAPI 直接作为静态文件服务，从而在生产环境中无需单独的开发服务器。
